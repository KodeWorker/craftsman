## Lessons from last iteration
1. Tool calls with multiple operations or params are not stable. Separate tool calls for each situation is much desired.
2. Workflows in tool description is not reliable. Make it a programmatic state machine for agent to follow rather than prompt.
3. Text insertion tool with wrong indents. If the text format is known (.py, .cpp, ... etc.), we could use liting for better results.
4. Not every tool call needs audition.
5. Plan/TODO tool: need more research on executor/verifier
6. client/server pair should be done sooner

## How to Work with Coding Agent
1. Human first; Agent code completion is in function level
2. Test coverage could be done by agent (happy path)
3. DO NOT OUTSOURCE THE THINKING
